# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in ReEnd-Components, please report it responsibly:

1. **Do NOT open a public issue** for security vulnerabilities
2. Send details to the maintainer via [GitHub private vulnerability reporting](https://github.com/VBeatDead/ReEnd-Components/security/advisories/new)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix release**: Depending on severity, typically within 2 weeks

## Scope

This policy applies to the ReEnd-Components npm package and documentation site. It does **not** cover:

- Dependencies managed by third parties (Radix UI, Tailwind CSS, etc.)
- User implementations or configurations

## Best Practices for Users

- Keep `reend-components` updated to the latest version
- Review the [CHANGELOG](https://github.com/VBeatDead/ReEnd-Components/releases) for security-related updates
- Report any suspicious behavior through the channels above
